## 🎯 一句话定位

**LogLens 填补 `tail -f | grep` 和 ELK 之间的巨大真空。** 它是一个基于 Tauri 的本地桌面日志分析工具——拖拽日志文件进去，自动解析格式、建立全文索引、提供强大的搜索过滤、并用 AI 帮你发现异常。安装包 < 50MB，启动 < 1 秒。

---

## 🕳️ 为什么还需要一个日志工具？

这是目前日志分析工具的残酷现状：

```
┌──────────────────────────────────────────────────┐
│                                                  │
│  tail -f / grep          ← 太原始、全靠人肉       │
│       🕳️                                      🕳️ │
│        🕳️   这里没有任何可用的桌面工具！    🕳️   │
│         🕳️                                  🕳️  │
│                                                  │
│  ELK Stack               ← 太重，本地起一套 4GB  │
│  Grafana Loki            ← 太重，配置复杂         │
│                                                  │
└──────────────────────────────────────────────────┘
```

| 现有方案 | 致命短板 |
|---|---|
| **glogg / klogg** | 已死/停滞 3 年，Qt 界面过时，无 AI |
| **lnav** | 终端 TUI，无 Web UI，学习曲线陡峭 (~10K Stars) |
| **Dozzle** | 只看 Docker 日志，不做持久化索引 (~13.5K Stars) |
| **chipmunk** | 还在 Rust 重写中，不知道何时发布 |
| **ELK / Loki** | 本地跑需要 4GB+ 内存，配置复杂，杀鸡用牛刀 |
| **Splunk / Datadog** | 商业化产品，天价，数据上云 |

**结论：桌面端日志分析工具已经「断代」了——一个活跃维护、现代化的竞品都没有。**

## ✨ 核心差异化功能

### 1. 拖拽即分析，零配置
- 📂 拖一个 `app.log`（500MB）进去 → 自动识别格式 → 秒级可搜索
- 支持格式：**JSON Lines、CSV、NGINX/Apache 访问日志、Syslog、纯文本、自定义正则**
- 自动发现时间戳字段、日志级别字段

### 2. 全文检索 + 结构化过滤
- 🔍 **全文搜索**（tantivy 引擎，百万行日志秒级检索）
- 🎯 **结构化过滤**：`level=ERROR AND service=api-gateway AND timestamp > 2025-01-01`
- 📊 **搜索统计**：匹配了多少条？分布在哪些时间段？Top 错误类型？

### 3. AI 异常检测（本地 Ollama，完全隐私）
- 🤖 **自动标出异常行**：不用写规则，AI 找出日志中的异常模式
- 🔗 **关联分析**：「这 500 条 Error 和前面那个部署事件有关吗？」
- 💬 **自然语言查询**：「Show me all timeout errors in the payment service」

### 4. 时间线 + 实时追踪
- ⏱️ **时间线视图**：像 Apache Skywalking 一样，按时间轴展示日志分布
- 📈 **统计面板**：每小时的 Error 数量趋势、Top 10 错误类型
- 🔴 **实时模式**：`tail -f` 的升级版——实时滚动 + 实时搜索过滤

### 5. 多文件关联分析
- 📁 同时打开 `nginx.log` + `app.log` + `db-slow.log`
- 🔗 **按时间对齐**：同一秒内跨文件的日志行排在一起
- 🔍 **关联搜索**：在 nginx 里点了某条 → 自动跳到 app.log 的对应时间

## 📁 支持的日志格式（开箱即用）

| 格式 | 自动识别 | 示例 |
|---|---|---|
| **JSON Lines** | ✅ | `{"time":"...","level":"ERROR","msg":"..."}` |
| **CSV** | ✅ | `2025-01-01,ERROR,api-gateway,timeout` |
| **NGINX/Apache** | ✅ | `192.168.1.1 - - [01/Jan/2025:12:00:00] "GET /api"` |
| **Syslog (RFC 5424)** | ✅ | `<134>1 2025-01-01T12:00:00Z myhost app 1234` |
| **纯文本** | ✅ (用户指定时间/级别正则) | 任意格式 |
| **自定义** | ✅ (正则模板) | 用户自定义解析规则 |
