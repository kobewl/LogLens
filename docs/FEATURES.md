# LogLens — 功能详细规格

## Phase 1：文件加载 & 基础展示（MVP）

### 1.1 文件加载（拖拽 & 选择）
```
┌──────────────────────────────────────────┐
│                                          │
│         📂 拖拽日志文件到这里             │
│                                          │
│          或 点击选择文件                   │
│                                          │
│  支持：.log .jsonl .csv .txt .gz         │
│  单文件最大：2GB                          │
└──────────────────────────────────────────┘
```

**功能点**：
- 拖拽文件到窗口 / 点击选择文件 / `File → Open`
- 支持格式：`.log` `.txt` `.jsonl` `.csv` `.gz`（自动解压）
- 自动检测编码（UTF-8 / UTF-16 / GBK / Latin-1）
- 加载进度条（解析 + 索引进度）
- 后台索引不阻塞 UI

### 1.2 格式自动识别
```
文件加载后弹出格式确认面板：

┌─ 格式确认 ────────────────────────────────┐
│ 检测到格式：JSON Lines                      │
│                                            │
│ 字段映射：                                  │
│  timestamp → @timestamp (ISO 8601)         │
│  level     → level (自动识别)               │
│  message   → message (自动识别)            │
│  service   → service                       │
│                                            │
│ [确认] [手动调整] [跳过(纯文本查看)]        │
└────────────────────────────────────────────┘
```

### 1.3 日志表格（虚拟滚动）
```
┌─ # ─┬─ Timestamp ───────┬─ Level ─┬─ Service ──┬─ Message ───────────┐
│ 1   │ 2025-01-01 12:00:00│ INFO    │ api-gateway│ Request received..  │
│ 2   │ 2025-01-01 12:00:01│ DEBUG   │ auth-svc   │ Token validated...  │
│ 3   │ 2025-01-01 12:00:01│ ERROR   │ payment-svc│ Connection timeout  │█
│ ... │ ...                │ ...     │ ...        │ ...                │█
│     │                    │         │            │                    │█
│     │                    │         │            │← 虚拟滚动，          │█
│     │                    │         │            │  仅渲染可视区域      │█
│ 999K│ 2025-01-02 08:00:00│ WARN    │ cache-svc  │ Cache miss...      │
└─────┴────────────────────┴─────────┴────────────┴────────────────────┘
```

**功能点**：
- **虚拟滚动**：100 万行日志流畅滚动（只渲染屏幕可见的 ~50 行）
- **列自适应**：根据格式自动显示字段列
- **列排序**：点击列头排序（按时间/级别/服务名）
- **级别颜色**：ERROR=红色、WARN=橙色、INFO=绿色、DEBUG=灰色
- **行选择**：单击选中行 → 右侧展开详情面板
- **多选**：Shift 多选行 → 复制/导出选中行
- **列显隐**：右键列头 → 选择显示/隐藏列

## Phase 2：全文索引 & 高级搜索

### 2.1 搜索栏
```
┌─ 搜索 ───────────────────────────────────────────┐
│ 🔍 [level=ERROR service=payment            ] [搜索]│
│                                                   │
│ 💡 语法提示:                                       │
│   field=value          精确匹配                    │
│   field:value          模糊匹配                    │
│   field>value          大于                       │
│   field<value          小于                       │
│   AND / OR / NOT       逻辑运算                    │
│   "exact phrase"       短语匹配                    │
│   regex:/pattern/      正则表达式                  │
│                                                   │
│ 示例: level=ERROR AND service:payment AND NOT ... │
└───────────────────────────────────────────────────┘
```

### 2.2 高级过滤构建器
```
┌─ 过滤条件 ────────────────────────────────────────┐
│                                                    │
│ ┌─ Condition 1 ──────────────────────────────┐    │
│ │ [level  ▼] [equals  ▼] [ERROR       ▼] [✕] │    │
│ └────────────────────────────────────────────┘    │
│                       AND                          │
│ ┌─ Condition 2 ──────────────────────────────┐    │
│ │ [service ▼] [contains ▼] [payment     ▼] [✕]│    │
│ └────────────────────────────────────────────┘    │
│                       AND                          │
│ ┌─ Condition 3 ──────────────────────────────┐    │
│ │ [timestamp▼][between ▼] [2025-01-01] [...]│ [✕] │
│ └────────────────────────────────────────────┘    │
│                                                    │
│ [+ 添加条件] [+ 添加组]                             │
└────────────────────────────────────────────────────┘
```

### 2.3 搜索结果
- 匹配行数 / 总行数 / 搜索耗时
- 匹配行在表格中**黄色高亮**
- 滚动条上显示**匹配密度标记**（像 VS Code 的 minimap）
- 搜索历史（最近 20 条搜索）

## Phase 3：AI 异常检测

```
用户点击「AI 分析」→ AI 扫描日志 → 标记异常行

结果：
┌─ AI 分析结果 ─────────────────────────┐

│                                        │
│ 🔴 #345,678  Connection timeout       │
│    异常原因：该错误在 12:00-12:05      │
│    之间集中爆发(127次)，之前1小时      │
│    内仅出现3次，可能是数据库短暂故障   │
│    置信度：高                          │
│                                        │
│ 🔴 #890,123  NullPointerException     │
```

### 3.2 自然语言查询
```
用户输入：「Show me all timeout errors in payment service after 3pm yesterday」

AI 转换为过滤条件：
  level=ERROR AND service=payment AND message:timeout AND timestamp>2025-01-06T15:00:00

然后自动执行搜索。
```

## Phase 4：多文件 & 实时追踪

### 4.1 多文件标签栏
```
┌─ [nginx-access.log] [app.log] [db-slow.log] [+] ─┐
```

- 同时打开多个文件，Tab 切换
- **关联视图**：两个文件按时间轴并排显示

### 4.2 实时追踪 (tail -f)
```
┌─ [● 实时追踪中...] ───────────────────────────────┐
│                                                    │
│  2025-01-01 12:00:05  INFO  新日志行实时追加...     │
│  2025-01-01 12:00:06  DEBUG ...                    │
│  2025-01-01 12:00:06  WARN  ...                    │
│                                                    │
│  [⏸ 暂停] [🔍 搜索过滤...]                         │
└────────────────────────────────────────────────────┘
```

- 监听文件变化（`notify` crate），实时追加新行
- 实时模式下搜索仍然有效（新行匹配搜索条件才显示）
- 暂停按钮（冻结当前视图，不丢数据）

### 4.3 告警规则
```
规则: level=ERROR 在 5 分钟内出现超过 10 次
动作: 系统通知 + 日志行高亮 + 声音提示
```
